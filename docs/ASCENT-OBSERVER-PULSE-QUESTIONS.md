## Question Design Principles

### Psychology Behind the Pulse

1. **Low Friction** — One question takes 15 seconds. Observers complete it.
2. **Behavioral, Not Personal** — Ask about observable actions, not personality.
3. **Growth-Oriented Language** — Frame change as normal and expected.
4. **Specific Enough to Recall** — Questions should trigger specific memories.
5. **Safe to Be Honest** — Anonymous responses encourage truth.

### Question Structure

Each focus area has:
- **Primary Question** — The core behavior measurement (always asked)
- **Scale** — 5-point behavioral frequency scale
- **Follow-up** — Optional open-ended question (asked 50% of the time)
- **Trend Questions** — Variations for months 2+ to measure change

## Core Focus Areas

### 1. Direct Feedback

**Primary Question (Month 1):**
> "In the past month, how often did you observe [Leader Name] giving clear, direct feedback when something needed to be addressed?"

**Scale:**
| Score | Label | Description |
|-------|-------|-------------|
| 1 | Never | I haven't seen them give direct feedback |
| 2 | Rarely | 1-2 times, and they seemed uncomfortable |
| 3 | Sometimes | 3-4 times, with varying directness |
| 4 | Often | 5+ times, usually clear and direct |
| 5 | Consistently | This is now typical behavior for them |

**Follow-up Questions (rotate):**
- "Can you recall a specific example of [Leader] giving direct feedback recently?"
- "What makes [Leader's] feedback effective or ineffective?"
- "Is there feedback you wish [Leader] had given but didn't?"

**Trend Question (Month 2+):**
> "Compared to last month, has [Leader]'s directness in giving feedback..."
- Decreased noticeably
- Stayed about the same  
- Improved somewhat
- Improved significantly
- Was already strong — maintained

## Reporting Logic

### Score Aggregation

```javascript
// For display to leader
const aggregateScore = (responses) => {
  if (responses.length < 3) return null; // Privacy threshold
  const avg = responses.reduce((a, b) => a + b, 0) / responses.length;
  return Math.round(avg * 10) / 10; // One decimal
};
```

### Trend Calculation

```javascript
// Month-over-month change
const calculateTrend = (currentScores, previousScores) => {
  const current = aggregateScore(currentScores);
  const previous = aggregateScore(previousScores);
  if (!current || !previous) return 'insufficient-data';
  const delta = current - previous;
  if (delta >= 0.5) return 'improving';
  if (delta <= -0.5) return 'declining';
  return 'stable';
};
```

### Display in UI

| Trend | Icon | Color | Message |
|-------|------|-------|---------|
| improving | ↑ | Green | "Your team sees improvement" |
| stable | → | Gray | "Holding steady" |
| declining | ↓ | Amber | "Opportunity to refocus" |
| insufficient-data | - | Gray | "Need 3+ responses to see trends" |

## Privacy & Ethics Considerations

### What Leaders See
- Aggregate scores (minimum 3 responses to show)
- Monthly trends
- Anonymous follow-up quotes (can opt to share)
- No individual response attribution

### What Leaders Do NOT See
- Individual scores
- Who responded vs. who didn't
- Response timestamps
- Whether a specific observer gave high/low scores

### Observer Experience
- One question max per month (per leader)
- Always quick — 15 seconds or less
- Can opt out at any time
- Reminded gently once if not completed
