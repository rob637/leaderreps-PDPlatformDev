# Cloud Functions — Copilot Instructions

## Overview
Firebase Cloud Functions in CommonJS. Mix of Gen 1 and Gen 2 functions.

## Rules
- **CommonJS only** — use `require()`, NOT `import`. This folder has its own package.json.
- Prefer Gen 2 syntax for new functions (`firebase-functions/v2`)
- All functions are defined in `index.js` (single entry point)
- Separate `npm install` required: `cd functions && npm install`
- Deploy: `firebase deploy --only functions` (from correct project context)

## Gen 2 Pattern
```js
const { onSchedule } = require('firebase-functions/v2/scheduler');
const { onCall, HttpsError } = require('firebase-functions/v2/https');

exports.myFunction = onCall({ region: 'us-central1' }, async (request) => {
  const { data, auth } = request;
  if (!auth) throw new HttpsError('unauthenticated', 'Must be logged in');
  // Implementation
});
```

## Gen 1 Pattern (legacy — avoid for new functions)
```js
const functions = require('firebase-functions/v1');

exports.myFunction = functions.https.onCall(async (data, context) => {
  if (!context.auth) throw new functions.https.HttpsError('unauthenticated', '...');
  // Implementation
});
```

## Key Existing Functions
- `scheduledDailyRollover` — daily cron for day transitions
- `manualRollover` — admin-triggered rollover
- `geminiProxy` — proxies AI API calls
- `scheduledNotificationCheck` — notification scheduling
- `validateInvitation` — invitation code validation

## Dependencies
- `firebase-admin` for Firestore access (already initialized)
- `firebase-functions` for function definitions
- Do NOT install packages in the root — use `functions/package.json`
