# Hooks — Copilot Instructions

## Overview
Custom React hooks encapsulate reusable logic. They bridge providers/services and components.

## Structure Template
```js
import { useState, useEffect, useMemo, useCallback } from 'react';
import { useAppServices } from '../services/useAppServices';

export const useMyHook = (options) => {
  const { services, serviceData } = useAppServices();
  const [loading, setLoading] = useState(true);

  const computedValue = useMemo(() => {
    // Derive from serviceData
  }, [serviceData]);

  const doAction = useCallback(async () => {
    // Call service methods
  }, [services]);

  return {
    data: computedValue,
    doAction,
    loading,
  };
};
```

## Rules
- Naming: `use` + PascalCase descriptor (`useDailyPlan`, `useCoachingRegistrations`)
- File naming: `use*.js` (NOT .jsx unless returning JSX — only `useAppServices.jsx` does)
- Always get data through `useAppServices()` or other context hooks — never raw Firestore
- Return an object (not array) with clearly named properties
- Use `useMemo` for computed/derived values
- Use `useCallback` for exposed functions
- Include `loading` in return when async data is involved
- Throw if used outside required provider: `if (!ctx) throw new Error('...')`

## Common Return Shapes
```js
// Data hooks
return { data, loading, error }

// Access control hooks
return { unlockedContent, unlockedCoaching, unlockedWeeks, loading }

// Daily plan hook
return { currentDayData, currentDayNumber, phase, phaseDayNumber, missedDays, toggleItemComplete, loading }

// Domain hooks
return { sessions, registrations, loading, error }
```
