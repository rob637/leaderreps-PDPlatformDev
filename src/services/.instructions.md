# Services — Copilot Instructions

## Overview
Services are the data/business logic layer between React components and Firestore. Components NEVER talk to Firestore directly.

## Patterns

### Stateless Services (most common)
Export named async functions. Always accept `db` as the first parameter:
```js
export const fetchSomething = async (db, userId, options) => {
  if (!db) throw new Error('Database not initialized');
  // Use firebase/firestore modular imports
};
```

### Stateful Services (createAppServices.js only)
Uses closure pattern with internal `stores` object, `listeners` array, and `cleanup()` method. Do NOT create new stateful services — extend `createAppServices.js` if needed.

### Service Object Pattern (membershipService.js style)
```js
export const myService = {
  doThing: (db, params) => { /* ... */ },
};
```

## Rules
- Import Firestore modular SDK: `import { doc, setDoc, query, ... } from 'firebase/firestore'`
- Use `onSnapshotEx`, `setDocEx`, `getDocEx` from `./firestoreUtils.js` for wrapped operations
- Use `buildModulePath()` from `./pathUtils.js` for Firestore paths
- Use `serverTimestamp()` for all timestamps — never `new Date()`
- Sanitize data from Firestore with `sanitizeTimestamps()` and `stripSentinels()`
- Constants use UPPER_SNAKE_CASE
- File naming: `*Service.js` for domain services, `*Utils.js` for utilities
- Register new services in `createAppServices.js` and expose via `AppServiceContext`
