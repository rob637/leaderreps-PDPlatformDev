# Widgets — Copilot Instructions

## Overview
Widgets are self-contained dashboard cards. Each widget handles its own data fetching and renders inside a `<Card>` wrapper.

## Structure Template
```jsx
import React, { useState, useEffect, useMemo, useCallback } from 'react';
import { IconName } from 'lucide-react';
import { Card } from '../ui';
import { useAppServices } from '../../services/useAppServices';
import { useDailyPlan } from '../../hooks/useDailyPlan';

const MyWidget = ({ helpText }) => {
  const { user, db, navigate, developmentPlanData } = useAppServices();
  const { currentDayData, cohortData } = useDailyPlan();

  // Local state for UI only
  const [loading, setLoading] = useState(true);

  // Data loading via useCallback + useEffect
  const loadData = useCallback(async () => {
    if (!user?.uid || !db) return;
    // fetch...
  }, [user?.uid, db]);

  useEffect(() => { loadData(); }, [loadData]);

  if (!currentDayData) return null;

  return (
    <Card title="Widget Title" icon={IconName} accent="TEAL" helpText={helpText}>
      {/* Content */}
    </Card>
  );
};

export default MyWidget;
```

## Rules
- Get data via `useAppServices()` and domain hooks — NEVER create Firestore listeners
- Accept `helpText` prop for info tooltips
- Use `<Card>` from `../ui` as the outer wrapper with `title`, `icon`, `accent` props
- Accent values: `"TEAL"`, `"ORANGE"`, `"NAVY"`
- Icons: import from `lucide-react` by name
- Navigation: `navigate('screen-key')` — no URLs
- Guard renders: return `null` if data not ready
- File naming: `*Widget.jsx` (PascalCase)
- Export: `export default WidgetName`

## Tailwind Patterns
- Container: `space-y-3`, `p-3`
- Text: `text-slate-700 dark:text-slate-200`, `text-sm`
- Backgrounds: `bg-slate-50 dark:bg-slate-800`
- Borders: `border border-slate-200 dark:border-slate-700 rounded-xl`
- Brand colors: `corporate-teal`, `corporate-orange`, `corporate-navy`
