# Screens — Copilot Instructions

## Overview
Screens are top-level route components, lazy-loaded via `ScreenRouter.jsx`. Each screen represents a full page the user navigates to.

## Adding a New Screen
1. Create `src/components/screens/MyScreen.jsx`
2. Add lazy import in `src/routing/ScreenRouter.jsx`
3. Add breadcrumb in `src/config/breadcrumbConfig.js` if needed
4. Navigate to it: `navigate('my-screen')` from any component


## Rules
- Export: `export default ScreenName`
- Navigation is string-based (`navigate('key')`) — NO react-router, NO URLs
- Screens often have tab navigation with `useState` for selected tab
- Use feature flags: `isFeatureEnabled('flag-name')` to gate sections
- Use access control hooks to check user permissions
- Break large screens into sub-components — extract render functions
- Screens may contain widgets and shared components
- No state passed via URL params — use contexts or service data
